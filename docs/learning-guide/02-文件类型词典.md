# 文件类型词典

这份文档是你的“翻译字典”。

以后你看到一个陌生文件，不要先怕，先判断它属于哪一种类型。

只要知道文件类型，通常就能先猜到它的大致职责。

---

## 先记住一句话

你现在最重要的，不是会写代码，而是会给文件“归类”。

一旦归类成功，理解难度会立刻下降。

---

## 前端常见文件类型

### `.vue`

它通常是页面或界面组件。

你可以把它理解成：

- 一个可见的页面
- 或页面上的一块界面拼图

在这个项目里的例子：

- `frontend/admin-web/src/views/LoginView.vue`
- `frontend/admin-web/src/views/DashboardView.vue`

你看 `.vue` 文件时，先问：

- 这个页面给谁用？
- 有哪些按钮、输入框、表格？
- 点了某个按钮以后会调用哪个 API？

### `router/*.ts`

这是页面入口地图。

你可以把它理解成“系统菜单和页面地址总表”。

在这个项目里的例子：

- `frontend/admin-web/src/router/index.ts`
- `frontend/admin-web/src/router/route-catalog.ts`

你看它时，先问：

- 页面路径是什么？
- 登录前能不能访问？
- 进入这个页面之前有没有权限判断？

### `api/modules/*.ts`

这是前端请求后端接口的封装。

你可以把它理解成“前台把申请单送到后台窗口的动作定义”。

在这个项目里的例子：

- `frontend/admin-web/src/api/modules/auth.ts`
- `frontend/admin-web/src/api/modules/core.ts`

你看它时，先问：

- 请求地址是什么？
- 是 GET 还是 POST？
- 传了哪些参数？
- 要求 token 吗？

### `stores/*.ts`

这是前端状态存放的地方。

你可以把它理解成“前端临时记账本”或“当前页面状态缓存”。

在这个项目里的例子：

- `frontend/admin-web/src/stores/financeCompany.ts`
- `frontend/admin-web/src/stores/financeWorkspace.ts`

### `components/*.vue`

这是可复用界面组件。

你可以把它理解成“页面会反复使用的小零件”，比如弹窗、表单片段、选择器。

## 后端常见文件类型

### `Controller`

这是后端接口入口。

你可以把它理解成“后台窗口接单员”。

在这个项目里的例子：

- `backend/auth-service/src/main/java/com/finex/auth/controller/AuthController.java`
- `backend/auth-service/src/main/java/com/finex/auth/controller/MvpController.java`

你看 Controller 时，先问：

- 它暴露了哪些接口？
- 这个接口给哪个前端页面用？
- 只是接收请求，还是里面直接写了很多业务判断？

### `Service`

这是业务规则层。

你可以把它理解成“真正决定怎么处理业务的人”。

在这个项目里的例子：

- `backend/auth-service/src/main/java/com/finex/auth/service/UserService.java`
- `backend/auth-service/src/main/java/com/finex/auth/service/impl/UserServiceImpl.java`

你看 Service 时，先问：

- 真正的业务规则是不是在这里？
- 它调用了哪些 Mapper？
- 它会不会同时改多个地方的数据？

### `Mapper`

这是数据库读写层。

你可以把它理解成“专门和数据库打交道的窗口”。

在这个项目里的例子：

- `backend/auth-service/src/main/java/com/finex/auth/mapper/UserMapper.java`
- `backend/auth-service/src/main/java/com/finex/auth/mapper/ProcessDocumentInstanceMapper.java`

你看 Mapper 时，先问：

- 它主要在查哪张表？
- 这次操作是查、增、改还是删？

### `Entity`

这是数据库表对应的对象。

你可以把它理解成“数据库一条记录在 Java 里的长相”。

在这个项目里的例子：

- `backend/auth-service/src/main/java/com/finex/auth/entity/User.java`

你看 Entity 时，先问：

- 对应哪张表？
- 主要字段有哪些？
- 哪些字段明显和业务有关，比如状态、金额、时间、用户、部门？

### `DTO`

通常是接口接收参数的对象。

你可以把它理解成“前端提交给后端的表单包裹”。

在这个项目里的例子：

- `backend/auth-service/src/main/java/com/finex/auth/dto/LoginDTO.java`
- `backend/auth-service/src/main/java/com/finex/auth/dto/ExpenseDocumentSubmitDTO.java`

你看 DTO 时，先问：

- 前端到底传了哪些字段？
- 有没有参数校验？

### `VO`

通常是后端返回给前端的数据对象。

你可以把它理解成“后端回传给页面看的结果包裹”。

在这个项目里的例子：

- `backend/auth-service/src/main/java/com/finex/auth/dto/LoginVO.java`
- `backend/auth-service/src/main/java/com/finex/auth/dto/UserProfileVO.java`

你看 VO 时，先问：

- 页面最终能拿到哪些字段？
- 这些字段够不够页面显示？

## 配置与脚本类文件

### `application.yml`

这是后端配置文件。

你可以把它理解成“系统运行参数表”。

在这个项目里的例子：

- `backend/auth-service/src/main/resources/application.yml`

它通常决定：

- 端口
- 数据库地址
- 用户名密码
- JWT 密钥
- 日志级别
- 异步任务参数

### `pom.xml`

这是 Maven 项目的依赖和模块说明。

你可以把它理解成“Java 项目的总装清单”。

在这个项目里的例子：

- `backend/pom.xml`
- `backend/auth-service/pom.xml`

### `package.json`

这是前端项目的依赖和命令清单。

你可以把它理解成“前端项目的总装清单和启动说明”。

在这个项目里的例子：

- `frontend/admin-web/package.json`

### `*.sql`

这是数据库脚本。

你可以把它理解成“建表、改表、补数据的施工单”。

在这个项目里的例子：

- `backend/sql/init.sql`
- `backend/sql/migrate_*.sql`

## 测试类文件

### `__tests__` 或 `*Test.java`

这是测试文件。

你可以把它理解成“规则保护网”。

在这个项目里的例子：

- `frontend/admin-web/src/router/__tests__/index.test.ts`
- `backend/auth-service/src/test/java/com/finex/auth/controller/AuthControllerTest.java`

你看测试时，先问：

- 它在保护哪条规则？
- 是怕谁以后改坏？
- 有没有覆盖关键分支，比如成功、失败、无权限、空参数？

## 一张最实用的归类表

| 你看到的文件 | 先把它理解成什么 | 你最先该问什么 |
| --- | --- | --- |
| `.vue` | 页面或界面块 | 谁在用它、点哪里会触发什么 |
| `router/*.ts` | 页面入口地图 | 从哪个地址进入、要不要权限 |
| `api/modules/*.ts` | 前端请求后端 | 调哪个接口、传什么参数 |
| `Controller` | 后端接单入口 | 前端是谁调它 |
| `Service` | 真正的业务规则 | 它改了什么业务状态 |
| `Mapper` | 数据库读写窗口 | 它碰哪张表 |
| `Entity` | 数据表的一行记录 | 关键业务字段有哪些 |
| `DTO` | 前端传入表单 | 收了哪些字段 |
| `VO` | 后端返回结果 | 页面最终拿到什么 |
| `application.yml` | 系统运行参数 | 端口、数据库、密钥怎么配 |
| `sql/*.sql` | 数据库施工单 | 建了什么表、改了什么结构 |
| `Test` | 保护规则的网 | 它在防哪种回归风险 |

## 你以后看到文件时的固定动作

以后你遇到陌生文件，按这 4 步来：

1. 先看文件路径。
2. 再判断它属于哪种文件类型。
3. 再问它上游是谁、下游是谁。
4. 最后再问它影响什么业务。

如果你直接一行一行读，很容易陷进去。

如果你先归类，再看上下游，理解速度会快很多。

---

## 本份文档的学习目标

学完后，你应该能自己说出：

- `.vue` 和 `Controller` 的区别是什么。
- `DTO` 和 `VO` 的区别是什么。
- `Service` 和 `Mapper` 的区别是什么。
- `application.yml` 和 `sql` 为什么都重要，但职责不同。
- `Test` 为什么不是“可有可无”的文件。
