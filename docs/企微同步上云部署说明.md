# 企微同步上云部署说明

这份说明对应当前仓库里已经完成的企微同步代码，目标是让“系统设置 -> 组织架构 -> 企微同步”在最短路径下稳定可用。

## 一、推荐上线形态

- 1 台 Linux 云服务器
- 1 个固定公网 IP
- 1 个正式域名
- 1 个企微自建应用
- 同域名反向代理方案：
  - 前端页面由 Nginx 托管
  - `/api/**` 反代到本机网关 `127.0.0.1:8080`
  - 前端继续使用默认 `VITE_API_BASE_URL=/api`

默认不采用“前端域名 + 独立 API 域名”方案，因为当前仓库不需要额外补 CORS 就能跑通同域部署。

## 二、仓库内已补齐的部署物料

- 后端生产环境模板：
  - `backend/.env.production.example`
- 前端生产环境模板：
  - `frontend/admin-web/.env.production.example`
- Nginx 配置模板：
  - `deploy/linux/nginx/finex.conf`
- systemd 服务模板：
  - `deploy/linux/systemd/finex-auth.service`
  - `deploy/linux/systemd/finex-gateway.service`
- Ubuntu 初始化脚本：
  - `deploy/linux/scripts/bootstrap_ubuntu.sh`
- 构建并发布脚本：
  - `deploy/linux/scripts/deploy_finex.sh`
- 安装运行环境脚本：
  - `deploy/linux/scripts/install_runtime.sh`

## 三、最短上线步骤

### 1. 准备云服务器

推荐最低配置：

- `2C2G`
- `40GB` 系统盘
- 固定公网 IP
- 安全组开放：
  - `22`
  - `80`
  - `443`

### 2. 准备域名

默认使用：

- `admin.your-domain.com`

将域名 A 记录解析到云服务器公网 IP。

### 3. 上传仓库并初始化服务器

在云服务器上执行：

```bash
git clone <你的仓库地址> /opt/finex/repo
cd /opt/finex/repo
sudo bash deploy/linux/scripts/bootstrap_ubuntu.sh
```

### 4. 准备生产环境变量

后端：

```bash
cp backend/.env.production.example backend/.env.production
```

至少修改这些字段：

- `FINEX_DB_URL`
- `FINEX_DB_USERNAME`
- `FINEX_DB_PASSWORD`
- `FINEX_JWT_SECRET`

前端：

```bash
cp frontend/admin-web/.env.production.example frontend/admin-web/.env.production
```

默认保留：

```env
VITE_API_BASE_URL=/api
```

### 5. 构建并发布

在服务器仓库根目录执行：

```bash
bash deploy/linux/scripts/deploy_finex.sh
```

该脚本会完成：

- Maven 打包后端
- npm 构建前端
- 将 jar 发布到 `/opt/finex/backend`
- 将前端静态资源发布到 `/opt/finex/www/admin`

### 6. 安装 Nginx 与 systemd 配置

先修改：

- `deploy/linux/nginx/finex.conf`

把：

- `admin.your-domain.com`

替换成你的正式域名。

然后执行：

```bash
sudo bash deploy/linux/scripts/install_runtime.sh
```

### 7. 配置 HTTPS

建议使用 Certbot：

```bash
sudo apt-get install -y certbot python3-certbot-nginx
sudo certbot --nginx -d admin.your-domain.com
```

签发成功后，再次检查：

```bash
sudo nginx -t
sudo systemctl restart nginx
```

## 四、企微后台配置

必须使用“自建应用”，不要使用“通讯录同步助手”的 Secret。

### 1. 自建应用配置

需要准备：

- `corpId`
- 自建应用 `Secret`
- 云服务器公网 IP
- 正式域名

### 2. 可信域名

在企微后台配置：

- `可信域名 = admin.your-domain.com`

### 3. 企业可信 IP

在企微后台配置：

- `企业可信 IP = 云服务器公网 IP`

以后企微同步请求都应从这台服务器发起，不再使用本地宽带动态 IP。

### 4. 应用权限与可见范围

确保该应用：

- 可以读取部门
- 可以读取成员
- 可见范围覆盖要同步的组织

## 五、系统设置中的企微参数填写

在系统设置 -> 组织架构 -> 同步连接配置中填写：

- `企业 ID` = `corpId`
- `通讯录 Secret` = 自建应用 `Secret`
- `AgentId（可选）` = 对应应用的 AgentId，可不填

保存后点击“立即同步”。

## 六、验证清单

### 1. 服务验证

```bash
systemctl status finex-auth
systemctl status finex-gateway
curl http://127.0.0.1:8080/api/auth/me
```

### 2. 出口 IP 验证

在服务器上执行：

```bash
curl https://api.ipify.org
```

返回结果必须与企微后台“企业可信 IP”一致。

### 3. 页面验证

- 能正常打开系统设置
- 能保存企微配置
- 点击“立即同步”后不再出现：
  - `60020`
  - `48009`

### 4. 数据验证

- `sys_department` 有企微部门
- `sys_user` 有企微员工
- 同步日志显示成功摘要

## 七、后续更新方式

以后代码有更新时，默认流程：

```bash
cd /opt/finex/repo
git pull
bash deploy/linux/scripts/deploy_finex.sh
sudo systemctl restart finex-auth finex-gateway
```

如果前端有改动，一并重启 Nginx：

```bash
sudo systemctl restart nginx
```
