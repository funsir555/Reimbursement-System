# 项目启动与初始化说明

## 1. 适用范围

本文档对应当前仓库真实状态，适用于本地开发、自测和新环境初始化。

当前默认启动的模块只有三个：

- `backend/auth-service`
- `backend/gateway`
- `frontend/admin-web`

## 2. 环境要求

- JDK 17
- Maven 3.9+
- Node.js 20+
- npm 10+
- MySQL 8.0

## 3. 环境变量

必须关注的变量如下：

- `FINEX_DB_URL`
- `FINEX_DB_USERNAME`
- `FINEX_DB_PASSWORD`
- `FINEX_JWT_SECRET`

常用可选变量：

- `FINEX_AUTH_SERVICE_PORT`
- `FINEX_GATEWAY_PORT`
- `FINEX_AUTH_SERVICE_URL`
- `FINEX_LOG_LEVEL`
- `FINEX_GATEWAY_LOG_LEVEL`
- `FINEX_MYBATIS_LOG_IMPL`
- `FINEX_ASYNC_CORE_POOL_SIZE`
- `FINEX_ASYNC_MAX_POOL_SIZE`
- `FINEX_ASYNC_QUEUE_CAPACITY`

变量示例见 `backend/.env.example`。

Windows 本地开发可以新建 `backend/.env.local.cmd`，并写入：

```bat
set FINEX_DB_PASSWORD=your-real-db-password
set FINEX_JWT_SECRET=your-long-random-secret-at-least-32-chars
```

`start-finex.bat` 会自动加载该文件，并在数据库密码缺失、JWT 仍是占位值或长度不足 32 个字符时直接拒绝启动。

## 4. 数据库初始化顺序

推荐在空库中按以下顺序执行：

1. `backend/sql/init.sql`
2. `backend/sql/init_custom_archive.sql`
3. `backend/sql/init_expense_type_tree.sql`
4. `backend/sql/init_async_task.sql`
5. `backend/sql/init_finance_gl.sql`
6. `backend/sql/init_process_flow_design.sql`

说明：

- `init.sql` 提供基础用户、组织、权限、MVP 相关表结构。
- `init_custom_archive.sql` 对应流程管理中的自定义档案。
- `init_expense_type_tree.sql` 对应流程管理中的费用类型树。
- `init_async_task.sql` 对应下载导出、验真、OCR、通知等异步任务。
- `init_finance_gl.sql` 对应财务凭证功能。
- `init_process_flow_design.sql` 对应流程设计器。

## 5. 升级脚本使用建议

如果不是新库，而是旧库升级，请先比对当前数据库状态，再按需执行：

- `backend/sql/migrate_company_id.sql`
- `backend/sql/migrate_custom_archive_codegen.sql`
- `backend/sql/migrate_custom_archive_seed_to_cn.sql`
- `backend/sql/migrate_department_global.sql`
- `backend/sql/migrate_system_settings.sql`
- `backend/sql/migrate_template_form_design_code.sql`
- `backend/sql/refresh_system_settings_permissions.sql`

建议原则：

- 新库优先走 `init*.sql`
- 旧库升级优先评估 `migrate*.sql`
- 权限或菜单不一致时再执行 `refresh*.sql`

## 6. 启动步骤

### 方式一：一键启动

在仓库根目录执行：

```bat
start-finex.bat
```

脚本会固定执行以下入口流程：

1. 加载 `backend/.env.local.cmd` 和可选的 `backend/.env.shadow.cmd`
2. 补齐默认端口与访问地址
3. 校验 `FINEX_DB_PASSWORD` 与 `FINEX_JWT_SECRET`
4. 先编译 `backend/common`
5. 委派 `start-finex.ps1` 处理旧进程、端口冲突和三个模块的实际启动

默认启动模块固定为：

- `backend/auth-service`
- `backend/gateway`
- `frontend/admin-web`

如果脚本提示端口冲突，需要先关闭占用 `8081`、`8080`、`5173` 的外部进程，或在 `backend/.env.local.cmd` 中改端口后再重试。

### 方式二：手动启动

先编译 `common`，确保 `auth-service` 本地运行时拿到最新公共类：

```bash
cd backend/common
mvn compile
```

启动 `auth-service`：

```bash
cd backend/auth-service
mvn spring-boot:run
```

启动 `gateway`：

```bash
cd backend/gateway
mvn spring-boot:run
```

启动前端：

```bash
cd frontend/admin-web
npm install
npm run dev
```

## 7. 启动后检查项

建议至少验证以下内容：

- 登录是否正常
- 首页是否能拉取当前用户和概览数据
- 发票列表是否能正常展示
- 流程管理页是否能加载模板和表单选项
- 异步任务接口是否能成功提交导出、验真、OCR 任务
- 财务凭证页是否能加载基础选项

## 8. 验证命令

后端：

```bash
cd backend
mvn test
```

前端单测：

```bash
cd frontend/admin-web
npm run test:unit
```

前端构建：

```bash
cd frontend/admin-web
npm run build
```
