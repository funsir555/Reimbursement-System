# 项目启动与初始化说明

更新日期：2026-04-12

## 1. 适用范围

本文档用于当前仓库的本地启动、环境初始化与基础验收说明。当前默认启动模块仍为：

- `backend/auth-service`
- `backend/gateway`
- `frontend/admin-web`

其中 `auth-service` 已承载 `auth / profile / process / async-task / voucher / settings / mvp / finance / expense / fixed-asset / archive-agent` 等多子域。

## 2. 当前验证基线

- backend：`mvn test` 通过，`303/303`
- frontend：`npm run test:unit` 通过，`211/211`

## 3. 环境要求

- JDK 17
- Maven 3.9+
- Node.js 20+
- npm 10+
- MySQL 8.0

## 4. 环境变量

关键变量：

- `FINEX_DB_URL`
- `FINEX_DB_USERNAME`
- `FINEX_DB_PASSWORD`
- `FINEX_JWT_SECRET`

常用可选变量：

- `FINEX_AUTH_SERVICE_PORT`
- `FINEX_GATEWAY_PORT`
- `FINEX_AUTH_SERVICE_URL`
- `FINEX_LOG_LEVEL`
- `FINEX_GATEWAY_LOG_LEVEL`
- `FINEX_GATEWAY_FRAMEWORK_LOG_LEVEL`
- `FINEX_MYBATIS_LOG_IMPL`
- `FINEX_ASYNC_CORE_POOL_SIZE`
- `FINEX_ASYNC_MAX_POOL_SIZE`
- `FINEX_ASYNC_QUEUE_CAPACITY`
- `FINEX_ASYNC_THREAD_NAME_PREFIX`
- `FINEX_ASYNC_AWAIT_TERMINATION_SECONDS`

样例文件：

- `backend/.env.example`
- `backend/.env.production.example`
- `frontend/admin-web/.env.production.example`

Windows 本地开发建议创建 `backend/.env.local.cmd`：

```bat
set FINEX_DB_PASSWORD=your-real-db-password
set FINEX_JWT_SECRET=your-long-random-secret-at-least-32-chars
```

说明：

- `start-finex.bat` 会自动加载 `backend/.env.local.cmd`
- `auth-service` 直接执行 `mvn spring-boot:run` 时，也会自动补读该文件
- 如果进程环境里已经显式设置了 `FINEX_*` 变量，则以进程环境值为准，不会被本地文件覆盖

## 5. 数据库初始化顺序

空库推荐按以下顺序执行：

1. `backend/sql/init.sql`
2. `backend/sql/init_custom_archive.sql`
3. `backend/sql/init_expense_type_tree.sql`
4. `backend/sql/init_async_task.sql`
5. `backend/sql/init_finance_gl.sql`
6. `backend/sql/init_process_flow_design.sql`

旧库升级请按实际差异执行 `migrate_*.sql` / `refresh_*.sql`，不要把全量初始化脚本当作日常增量升级脚本。

## 6. 启动步骤

### 方式一：一键启动

```bat
start-finex.bat
```

固定流程：

1. 加载 `backend/.env.local.cmd` 与可选的 `backend/.env.shadow.cmd`
2. 校验数据库密码与 JWT secret
3. 先编译 `backend/common`
4. 委派 `start-finex.ps1` 处理端口冲突和三个模块的实际启动

默认端口：

- `8081`：auth-service
- `8080`：gateway
- `5173`：frontend/admin-web

### 方式二：手动启动

先编译 `common`：

```bash
cd backend/common
mvn compile
```

启动后端：

```bash
cd ../auth-service
mvn spring-boot:run

cd ../gateway
mvn spring-boot:run
```

启动前端：

```bash
cd ../../frontend/admin-web
npm install
npm run dev
```

## 7. 启动后检查项

建议至少验证：

- 登录是否正常
- 首页是否能拉取当前用户与概览数据
- 报销、流程、财务、档案、固定资产等页面是否能拿到基础元数据
- 如果此前前端报 500，先检查 `auth-service` 与 `gateway` 是否已成功启动

## 8. 常用验证命令


前端单测：
