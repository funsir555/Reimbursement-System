# 项目启动与初始化说明

更新时间：2026-04-11

## 1. 适用范围

本文档对应当前仓库真实状态，适用于本地开发、自测、部署前初始化和环境对齐。

当前默认启动模块仍只有三个：

- `backend/auth-service`
- `backend/gateway`
- `frontend/admin-web`

但要特别注意：当前 `auth-service` 已承载 auth、profile、process、async-task、voucher、settings、mvp、finance、expense、fixed-asset、archive-agent 等多个子域。

## 2. 当前验证基线

- backend：`mvn test` 通过，`297/297`
- frontend：`npm run test:unit` 通过，`211/211`

如果只是初始化环境，默认不要求立刻复跑全部用例；但任何治理批次或较大改动完成后，都应回到这套基线。

## 3. 环境要求

- JDK 17
- Maven 3.9+
- Node.js 20+
- npm 10+
- MySQL 8.0

## 4. 环境变量

必须关注的变量：

- `FINEX_DB_URL`
- `FINEX_DB_USERNAME`
- `FINEX_DB_PASSWORD`
- `FINEX_JWT_SECRET`

常用可选变量：

- `FINEX_AUTH_SERVICE_PORT`
- `FINEX_GATEWAY_PORT`
- `FINEX_AUTH_SERVICE_URL`
- `FINEX_LOG_LEVEL`
- `FINEX_GATEWAY_LOG_LEVEL`
- `FINEX_GATEWAY_FRAMEWORK_LOG_LEVEL`
- `FINEX_MYBATIS_LOG_IMPL`
- `FINEX_ASYNC_CORE_POOL_SIZE`
- `FINEX_ASYNC_MAX_POOL_SIZE`
- `FINEX_ASYNC_QUEUE_CAPACITY`
- `FINEX_ASYNC_THREAD_NAME_PREFIX`
- `FINEX_ASYNC_AWAIT_TERMINATION_SECONDS`

样例文件：

- `backend/.env.example`
- `backend/.env.production.example`
- `frontend/admin-web/.env.production.example`

Windows 本地开发建议新建 `backend/.env.local.cmd`：

```bat
set FINEX_DB_PASSWORD=your-real-db-password
set FINEX_JWT_SECRET=your-long-random-secret-at-least-32-chars
```

`start-finex.bat` 会自动加载该文件，并在数据库密码缺失、JWT 仍是占位值或长度不足 32 个字符时直接拒绝启动。

## 5. 数据库初始化顺序

空库推荐顺序：

1. `backend/sql/init.sql`
2. `backend/sql/init_custom_archive.sql`
3. `backend/sql/init_expense_type_tree.sql`
4. `backend/sql/init_async_task.sql`
5. `backend/sql/init_finance_gl.sql`
6. `backend/sql/init_process_flow_design.sql`

说明：

- `init.sql`：基础用户、组织、权限、MVP 相关表
- `init_custom_archive.sql`：流程管理的自定义档案
- `init_expense_type_tree.sql`：费用类型树
- `init_async_task.sql`：导出、验真、OCR、通知等异步任务
- `init_finance_gl.sql`：财务、总账、档案相关表
- `init_process_flow_design.sql`：流程设计器

旧库升级请按需执行 `migrate_*.sql` / `refresh_*.sql`，不要把整库初始化脚本当成日常升级脚本。

## 6. 启动步骤

### 方式一：一键启动

```bat
start-finex.bat
```

脚本固定流程：

1. 加载 `backend/.env.local.cmd` 和可选的 `backend/.env.shadow.cmd`
2. 补齐默认端口与访问地址
3. 校验 `FINEX_DB_PASSWORD` 与 `FINEX_JWT_SECRET`
4. 先编译 `backend/common`
5. 委派 `start-finex.ps1` 处理旧进程、端口冲突和 3 个模块的实际启动

默认端口：

- `8081`：auth-service
- `8080`：gateway
- `5173`：frontend/admin-web

### 方式二：手动启动

先编译 `common`：

```bash
cd backend/common
mvn compile
```

启动后端：

```bash
cd ../auth-service
mvn spring-boot:run

cd ../gateway
mvn spring-boot:run
```

启动前端：

```bash
cd ../../frontend/admin-web
npm install
npm run dev
```

## 7. 启动后检查项

建议至少验证：

- 登录是否正常
- 首页是否能拉取当前用户和概览数据
- 报销/发票列表是否正常返回
- 流程管理页是否能加载模板与表单选项
- 财务、档案、固定资产等页面是否能拿到基础元数据
- archive-agent 与 expense-voucher 相关入口是否无 401/403/500 漂移

## 8. 常用验证命令

后端：

```bash
cd backend
mvn test
```

前端单测：

```bash
cd frontend/admin-web
npm run test:unit
```

前端构建：

```bash
cd frontend/admin-web
npm run build
```
