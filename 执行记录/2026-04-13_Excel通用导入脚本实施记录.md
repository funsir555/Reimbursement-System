# 2026-04-13 Excel 通用导入脚本实施记录

## 1. 变更摘要

- 新增脚本：`backend/sql/import_excel_to_mysql.py`
- 目标：提供一个可复用的 `.xlsx` -> MySQL 导入工具，默认读取 `application.yml` 中的数据源配置。
- 导入策略：Excel 表头必须使用数据库字段名；重复键直接报错停止；正式导入前建议先执行 `--dry-run`。

## 2. 已实现能力

- 读取 `backend/auth-service/src/main/resources/application.yml` 中的 `spring.datasource` 配置
- 兼容 `${ENV:default}` 占位写法
- 读取指定工作表或默认第一张工作表
- 将首行作为数据库字段名
- 校验目标表是否存在、字段是否匹配、必填列是否缺失
- 按 `--key-columns` 做 Excel 内部重复检查与数据库重复检查
- 自动识别 `DATE` / `DATETIME` / `TIMESTAMP` 列并做标准日期校验
- 自动识别 `DECIMAL(18,2)` 列并做金额校验
- 支持 `--date-columns` / `--amount-columns` / `--text-columns` 追加强制校验列
- 检测 `?`、`????`、`锟`、`�` 等明显乱码文本
- 参数化写库，逐批导入，失败时回滚

## 4. 使用说明摘要

- 依赖安装：
  - `pip install openpyxl pymysql`
- 示例命令：
  - `python backend/sql/import_excel_to_mysql.py --file C:\data\demo.xlsx --table gl_Customer --key-columns cCusCode --dry-run`
- 关键约束：
  - 日期列必须为标准日期格式
  - 金额列按 `DECIMAL(18,2)` 标准校验
  - 中文文本不能出现明显乱码
  - 已存在键值直接报错停止，不覆盖旧数据
