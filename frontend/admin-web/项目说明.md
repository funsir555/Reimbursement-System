# 财务报销系统前端项目说明

> 本文档面向没有前端开发经验的开发者，帮助你理解项目结构和每个文件的作用。

## 📁 项目整体结构

```
frontend/admin-web/          # 前端项目根目录
├── public/                  # 静态资源文件夹（直接复制到服务器，不经过构建）
│   └── favicon.ico         # 网站图标
├── src/                     # 源代码文件夹（核心代码都在这里）
│   ├── assets/             # 项目资源（图片、字体等）
│   ├── components/         # 公共组件（可复用的UI组件）
│   ├── layouts/            # 页面布局（整体页面框架）
│   ├── router/             # 路由配置（页面跳转管理）
│   ├── styles/             # 样式文件（CSS样式）
│   ├── views/              # 页面视图（具体页面）
│   ├── App.vue             # 根组件（整个应用的入口）
│   └── main.ts             # 入口文件（程序启动的地方）
├── index.html              # HTML模板（浏览器看到的第一个文件）
├── package.json            # 项目配置（依赖包列表）
├── tsconfig.json           # TypeScript配置
├── vite.config.ts          # Vite构建工具配置
├── tailwind.config.js      # Tailwind CSS配置
└── postcss.config.js       # PostCSS配置
```

### 二、源代码文件夹 `src/`

这是你最需要关注的地方！

#### 1. `main.ts` - 程序的入口

**文件作用：**
就像一栋大楼的大门，是所有代码开始执行的地方。

**代码解读：**
```typescript
import { createApp } from 'vue'           // 引入Vue框架
import ElementPlus from 'element-plus'    // 引入UI组件库
import App from './App.vue'               // 引入根组件
import router from './router'             // 引入路由配置

const app = createApp(App)                // 创建Vue应用实例
app.use(router)                           // 使用路由
app.use(ElementPlus)                      // 使用UI组件库
app.mount('#app')                         // 挂载到HTML的#app元素
```

**你可以做什么：**
- 在这里添加全局使用的插件
- 配置全局错误处理
- 添加全局样式

---

#### 2. `App.vue` - 根组件

**文件作用：**
这是所有页面的"外壳"，最外层的容器。

**代码结构：**
```vue
<template>
  <!-- 这里放HTML结构 -->
  <router-view />   <!-- 这是关键：显示当前路由对应的页面 -->
</template>

<script setup lang="ts">
  // 这里放JavaScript逻辑
</script>

<style>
  /* 这里放CSS样式 */
</style>
```

**三个部分：**
| 部分 | 作用 | 语言 |
|------|------|------|
| `<template>` | 定义页面的HTML结构 | HTML +
Vue指令 |
| `<script>` | 处理数据和逻辑 | TypeScript |
| `<style>` | 定义样式 | CSS |

---

#### 3. `router/index.ts` - 路由配置

**文件作用：**
管理"URL地址"和"页面组件"的对应关系。

**代码解读：**
```typescript
const routes = [
  {
    path: '/login',           // URL地址
    name: 'login',            // 路由名称
    component: LoginView      // 对应的页面组件
  },
  {
    path: '/dashboard',       // URL地址
    name: 'dashboard',
    component: () => import('../views/DashboardView.vue')  // 懒加载
  }
]
```

**路由类型：**
| 类型 | 写法 | 使用场景 |
|------|------|----------|
| 直接引入 | `component: LoginView` | 首页等必须快速加载的页面 |
| 懒加载 | `component: () => import('...')` | 其他页面，减少首屏加载时间 |

**你可以做什么：**
- 添加新页面时在这里注册路由
- 配置路由守卫（如登录检查）

### 三、页面文件夹 `src/views/`

这里存放所有具体的页面。

#### 1. `LoginView.vue` - 登录页面

**文件作用：**
用户登录的页面，左右分栏设计。

**学习要点：**
```vue
<!-- 数据绑定：: 表示绑定数据 -->
<el-input v-model="loginForm.username" />
<!-- v-model 实现了输入框和数据的自动同步 -->

<!-- 事件绑定：@ 表示绑定事件 -->
<el-button @click="handleLogin">登录</el-button>
<!-- 点击按钮时执行 handleLogin 函数 -->

<!-- 条件渲染：v-if 控制显示/隐藏 -->
<div v-if="loginType === 'password'">密码登录表单</div>
<div v-else>验证码登录表单</div>
```

**Vue 指令速查：**
| 指令 | 作用 | 示例 |
|------|------|------|
| `v-model` | 双向数据绑定 | `<input v-model="name">` |
| `v-if` | 条件渲染 | `<div v-if="show">` |
| `v-for` | 循环渲染 | `<div v-for="item in list">` |
| `@click` | 点击事件 | `<button @click="fn">` |
| `:class` | 动态类名 | `<div :class="{active: isActive}">` |

### 四、组件文件夹 `src/components/`

**什么是组件？**
组件是可复用的UI块，就像乐高积木。同一个组件可以在多个页面使用。

#### `NewExpenseDialog.vue` - 新建报销弹窗

**文件作用：**
点击"新建报销"按钮后弹出的表单对话框。

**为什么做成组件？**
因为这个弹窗可能在多个地方被调用（首页、报销列表页等）。

**关键概念 - 父子组件通信：**
```vue
<!-- 父组件中使用 -->
<new-expense-dialog v-model="showDialog" />

<!-- 子组件中定义 -->
const props = defineProps<{
  modelValue: boolean    // 接收父组件传入的值
}>()

const emit = defineEmits<{
  (e: 'update:modelValue', value: boolean): void  // 通知父组件更新
}>()
```

| 方向 | 方式 | 说明 |
|------|------|------|
| 父 → 子 | props | 父组件通过属性传递数据 |
| 子 → 父 | emit | 子组件触发事件通知父组件 |

## 🛠 如何修改和扩展

### 场景1：修改主题色

编辑 `src/styles/main.css`：
```css
:root {
  --el-color-primary: #你的颜色代码;
}
```

### 场景2：添加新页面

1. **创建页面文件** `src/views/NewPageView.vue`
2. **注册路由** `src/router/index.ts`
   ```typescript
   {
     path: '/new-page',
     name: 'new-page',
     component: () => import('../views/NewPageView.vue')
   }
   ```
3. **添加菜单项** `src/layouts/MainLayout.vue`
   ```vue
   <el-menu-item index="/new-page">
     <el-icon><NewIcon /></el-icon>
     <template #title>新页面</template>
   </el-menu-item>
   ```

### 场景3：修改表格列

编辑对应的列表页面，如 `expense/ExpenseListView.vue`：
```vue
<el-table-column prop="新字段" label="新列名" width="120" />
```

### 场景4：添加表单字段

编辑表单组件，如 `NewExpenseDialog.vue`：
```vue
<el-form-item label="新字段">
  <el-input v-model="form.newField" />
</el-form-item>
```

## 🆘 常见问题

### Q1: 运行 `npm install` 报错？
**A:** 检查是否安装了Node.js，版本需要18以上。
```bash
node -v    # 查看版本
```

### Q2: 如何调试代码？
**A:** 
1. 运行 `npm run dev` 启动项目
2. 按 F12 打开浏览器开发者工具
3. 在 Sources 面板找到代码，打断点调试

### Q3: 修改后页面没变化？
**A:**
1. 检查是否保存了文件
2. 查看终端是否有报错
3. 尝试刷新浏览器页面

### Q4: 如何添加图标？
**A:**
项目使用 Element Plus 图标，在 https://element-plus.org/zh-CN/component/icon.html 查找图标名，然后：
```vue
<el-icon><IconName /></el-icon>
```
