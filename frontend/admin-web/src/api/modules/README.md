# API modules 说明

`src/api/modules/` 是前端 API 封装与类型定义的主目录。

## 目录角色
- `*.ts` ：按业务域封装接口请求。
- `*-types.ts` ：定义页面提交和接口返回的数据结构。
- `core.ts` ：提供统一请求、超时、token 和下载能力。
- `shared.ts` ：统一导出各业务域类型。

## 阅读建议
1. 先看对应的 `views/*.vue` ，明确页面在做什么。
2. 再看对应的 `modules/*.ts` ，确认页面会调哪些接口。
3. 最后对照 `*-types.ts` ，弄清楚请求参数和返回数据结构。

## 使用原则
- 尽量保持请求路径、参数名和类型定义一致。
- 新增业务域时，优先按现有目录结构增加接口文件和类型文件。
- 复杂请求优先在本目录内做注释，避免页面层直接堆破碎逻辑。
